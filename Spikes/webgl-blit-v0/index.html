What this measures: the cost of gl.texImage2D(TEXTURE_2D, 0, RGBA, RGBA, UNSIGNED_BYTE, sourceCanvas) when source and destination are different WebGL2 contexts on different canvases. This is the load-bearing question for SkiaMonoGameRendering's WebGL/KNI option D.

CPU time is the wall-clock cost of the texImage2D call (synchronous JS-side work + IPC enqueue, no GPU wait). GPU time is the actual GPU work via EXT_disjoint_timer_query_webgl2 if available, reported with the precision the browser allows. Results lag a few frames (queries are async).

Success criteria: at 1920×1080, GPU time should be sub-millisecond on a mid-range GPU in Chrome, and CPU time should be a few hundred microseconds at most. If either is multi-millisecond, this path is too expensive and we revisit Option A.

Upload paths:

  • texImage2D(canvas) — the default cross-context path. Fast on Chrome/Edge (Blink + ANGLE/D3D11), reportedly very slow on Firefox (likely a CPU readback + re-upload internally).
  • texSubImage2D(canvas) — avoids re-allocating destination storage each frame; otherwise identical to the above.
  • createImageBitmap → texImage2D(bitmap) — async path. Firefox has had GPU-accelerated ImageBitmap handling for a while; the explicit "this is going to a texture" hint may dodge the readback heuristic. Total CPU time here includes the createImageBitmap cost.
  • OffscreenCanvas + transferToImageBitmap → texImage2D(bitmap) — the ownership-transfer fast path. Source becomes an OffscreenCanvas; transferToImageBitmap hands off its GPU pixels synchronously and zero-copy on supporting browsers.
  • readPixels → texImage2D(Uint8Array) — the Option B baseline. Pulls pixels to CPU via readPixels on source context, uploads as a typed array to destination. If this matches the "slow" cross-context path on a given browser, you've confirmed that browser is already doing readback internally for the canvas upload.

Other toggles: flipping UNPACK_PREMULTIPLY_ALPHA_WEBGL or UNPACK_FLIP_Y_WEBGL can push the browser off the direct-blit fast path and into a conversion-shader pass. Flush forces ordering between source and destination contexts.
